# Support

## Where to ask

- Use [GitHub Issues](https://github.com/oaslananka/infra-lens-mcp/issues) for reproducible bugs, release blockers, packaging problems, CI failures, and actionable feature requests.
- Use [GitHub Discussions](https://github.com/oaslananka/infra-lens-mcp/discussions) for usage questions, deployment advice, and ideas that are not ready for implementation.
- Use [SECURITY.md](./SECURITY.md) for vulnerabilities. Do not report vulnerabilities in public issues or discussions.

## Response expectations

Maintainer availability is best effort. Security reports are expected to receive an initial response within 72 hours. Public issues are triaged by priority:

- `priority:P0`: target initial response within 1 business day.
- `priority:P1`: target initial response within 3 business days.
- `priority:P2`: target initial response within 7 business days.
- `priority:P3`: no fixed response target.

## Scope

Support covers the current default branch and the latest published package. Older versions are handled only when the same issue affects the current release line.

## Before opening an issue

- Search existing issues and pull requests.
- Remove secrets, hostnames, tokens, private keys, and private infrastructure details.
- Include Node.js version, transport (`stdio` or HTTP), package version, and the relevant redacted command or tool call.
- Use issues for work that can be verified and closed. Use discussions when the request is still exploratory.
