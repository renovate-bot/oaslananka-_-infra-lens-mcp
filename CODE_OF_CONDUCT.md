# Code of Conduct

## Our standard

This project is for practical infrastructure and security-sensitive operations work. Participants are expected to be respectful, precise, and collaborative.

Examples of welcome behavior:

- Use clear, technical language and assume good intent.
- Keep incident and security discussions focused on evidence and impact.
- Redact secrets, credentials, hostnames, and private operational details before posting.
- Accept maintainer decisions when scope, safety, or release risk requires a narrower path.

Examples of unacceptable behavior:

- Harassment, insults, threats, or sustained disruptive behavior.
- Posting secrets, private keys, access tokens, or private infrastructure details.
- Publicly disclosing a vulnerability before the security process has had time to respond.
- Pressuring maintainers to bypass security, CI, release, or verification gates.

## Enforcement

Maintainers may edit, hide, lock, or remove comments, issues, pull requests, or discussion threads that violate this policy. Repeated or severe violations may result in a temporary or permanent ban from repository participation.

## Reporting

Report conduct concerns to **oaslananka@gmail.com**. For vulnerabilities, use the process in [SECURITY.md](./SECURITY.md) instead of a public issue.
